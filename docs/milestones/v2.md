## Observability

Full distributed observability across API and web using **OpenTelemetry**. The goal is a single trace ID that connects a browser interaction through the API, the database, and the email service.

- [ ] OpenTelemetry SDK setup ‚Äî API (`@opentelemetry/sdk-node`) and web (`@opentelemetry/sdk-web`)
- [ ] API: HTTP request tracing (Express instrumentation ‚Äî spans per route, status, latency)
- [ ] API: Prisma query tracing (`@prisma/instrumentation` or manual spans around DB calls)
- [ ] API: Pino log correlation ‚Äî inject `trace_id` and `span_id` into every log entry so logs and traces are joinable in the backend
- [ ] Web: client-side error tracking (uncaught exceptions, error boundary captures forwarded via OTLP or a thin fetch call)
- [ ] Web: Core Web Vitals and custom performance spans
- [ ] Choose and configure an OTLP collector/backend (preference: Grafana Cloud or Honeycomb ‚Äî evaluate at implementation time)
- [ ] Replace the V1 client no-op logger with an OTLP-backed implementation; keep the same `logger` interface so call sites need no changes
